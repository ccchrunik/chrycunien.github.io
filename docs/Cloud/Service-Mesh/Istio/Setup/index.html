Istio Setup

Envivronmentâ€‹

  • minikube: v1.23.0
  • Kubernetes: v1.22.1
  • Istio: v1.11.4

Install Istioâ€‹

  1. Install Istio
$ curl -L https://istio.io/downloadIstio | sh -
  1. Change into the istio directory
$ cd istio-1.11.4
  1. Update the PATH environment variable (you can add the path in ~/.zshrc or ~/.bashrc as well)
$ export PATH=$PWD/bin:$PATH
  1. Install a demo Istio profile (the basic structure of Istio in the cluster, you can select different profile)
$ istioctl install --set profile=demo -y
  1. Verify installation.
$ istioctl verify-install

Application Setupâ€‹

  1. Enable istio-injection label
$ kubectl label namespace default istio-injection=enabled
  1. Install the application
$ kubectl apply -f samples/bookinfo/platform/kube/bookinfo.yaml
  1. Check whether each pod has two containers (one is the original container, the other is the sidecar)
$ kubectl get pods -A
  1. Verify if the application is running
$ kubectl exec "$(kubectl get pod -l app=ratings -o jsonpath='{.items[0].metadata.name}')" -c ratings -- curl -sS productpage:9080/productpage | grep -o "<title>.*</title>"
  • Note: If something doesn't run correctly, use istioctl analyze to find out the problem

Enable Ingress trafficâ€‹

  1. Create a gateway (a crd)
$ kubectl apply -f samples/bookinfo/networking/bookinfo-gateway.yaml
  1. Get istio ingress port and host
export INGRESS_PORT=$(kubectl -n istio-system get service istio-ingressgateway -o jsonpath='{.spec.ports[?(@.name=="http2")].nodePort}')
export SECURE_INGRESS_PORT=$(kubectl -n istio-system get service istio-ingressgateway -o jsonpath='{.spec.ports[?(@.name=="https")].nodePort}')
export INGRESS_HOST=$(minikube ip)
export GATEWAY_URL=$INGRESS_HOST:$INGRESS_PORT
  1. Send traffic to istio ingress gateway
# Method 1
$ minikube tunnel

# Method 2
$ minikube addons enable ingress
  1. Verify the page
# Method 1
$ curl "http://$GATEWAY_URL/productpage"

# Method 2
# Copy "http://$GATEWAY_URL/productpage" to the browser

Add Custom Traffic Rulesâ€‹

  1. Add desination rules (the virtual services had been defined in bookinfo-gateway.yaml)
$ kubectl apply -f samples/bookinfo/networking/destination-rule-all.yaml

# If using tls
$ kubectl apply -f samples/bookinfo/networking/destination-rule-all-mtls.yaml
  1. After deployed, you will find some errors, it's fine in that it defines some rules that are not used in addition.

Dashboardâ€‹

Istio provides a Kiali dashboard to better understand the network view.

  1. Install addons (note: these addons are just for demo, not for production)
$ kubectl apply -f samples/addons
  1. Wait for them to be deployed
$ kubectl rollout status deployment/kiali -n istio-system
$ kubectl -n istio-system get svc kiali
  1. View the dashbaord
$ istioctl dashboard kiali
  1. Continue sending traffic to our application to gather metrics
$ while sleep 0.01; do curl -sS "http://$GATEWAY_URL/productpage" &> /dev/null ; done
  1. Select the graph section on the sidebard (you may have to reload several times)

Cleanupâ€‹

  1. Remove application-related resources (including gateway, virtual service, ...)
$ ./samples/bookinfo/platform/kube/cleanup.sh
  1. Remove istio-related resources
# Remove addons
kubectl delete -f samples/addons

# Remove namespace and injection label
kubectl delete namespace istio-system
kubectl label namespace default istio-injection-
